What is an API?

Before moving on to the Marvel Comics API, let's learn more about APIs.

API stands for Application Programming Interface. It's a fancy sounding word, but it's a relatively simple concept. Essentially, an API
serves as an interface between software programs and facilitates their interaction. Think about how two people who speak different
languages can communicate with each other. Well, they would need a translator right? The translator, whether it is another person or a device,
serves as an interface for the two people to communicate. In this case, the translator is the API!

So, what are we going to do with the Marvel Comic API? As you are probably aware, Marvel is big. They have been around since the 70's and their
library is HUGE. We will be using the API to make calls to their resources to learn more about their vast roster of superheroes.

Making a Call

Now that we know what an API is let's make a call to the Marvel Comics API.

Currently, the Marvel API accept only HTTP GET requests. This means that we have to use the keyword GET when we open a request to
the endpoint https://gateway.marvel.com/v1/public/comics. An endpoint is a unique URL used to communicate with an API. In this case, we are
making a call to the comics endpoint. This endpoint will return full views of comics in Marvel's library.

The true argument in the script states that we are making an asynchronous call. That means that we must add an event listener that will
wait until the request loads and then parse the response and log it in the console. If the request fails, then an error is logged. The last part of the script,
request.send(null); simply states that we are not sending any additional data to our request.
